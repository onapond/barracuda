# Barracuda Designer Content Request

이 문서는 Barracuda 홈페이지 MVP에 실제 콘텐츠를 반영하기 위해
디자이너 또는 브랜드 담당자가 전달해야 하는 텍스트와 이미지 목록을 정리한 문서다.

목표:
- 현재 placeholder 콘텐츠를 실제 운영용 콘텐츠로 교체
- 디자인/개발이 같은 기준으로 파일명과 문구를 정리
- 전달받은 자료를 바로 `data/site-content.ts`와 `public/images`에 반영 가능하게 준비

---

## 1. 전달 원칙

- 텍스트는 최종본 또는 최종에 가까운 버전으로 전달
- 이미지 파일은 실제 사용 가능한 원본 또는 고해상도 압축본으로 전달
- 파일명은 아래 권장 파일명을 맞추는 것을 우선
- 미정 정보는 `TBD`로 표기
- 주소, 운영시간, 연락처, 인스타그램 링크는 실제 운영 기준으로 전달

## 3. 이미지 요청 목록

아래 이미지를 전달받아야 한다.

- Hero 대표 이미지 1장
- Brand 이미지 3장
- Coffee 이미지 1장
- Space 이미지 3장
- Store 이미지 3장

총 11장

---

## 4. 권장 파일명

가능하면 아래 파일명으로 전달한다.

```txt
hero.jpg
brand-1.jpg
brand-2.jpg
brand-3.jpg
coffee-1.jpg
space-1.jpg
space-2.jpg
space-3.jpg
store-1.jpg
store-2.jpg
store-3.jpg
```

PNG를 써야 하는 경우에도 이름 규칙은 그대로 유지한다.

예:
- `hero.png`
- `space-2.png`

---

## 5. 이미지 가이드

- 메뉴판 느낌보다 공간, 무드, 질감, 브랜드 인상 중심
- 과한 보정보다 차분한 톤 우선
- 너무 밝게 날아간 이미지보다 깊이 있는 톤 우선
- 공간 사진은 실제 머무는 분위기가 느껴져야 함
- 커피 사진은 제품 정보보다 로스터리 기반 브랜드 감도 전달이 우선
- Store 이미지는 실제 쇼핑몰 상세컷보다 브랜드 제품 프리뷰 느낌이 적합

---

## 6. 이미지 품질 기준

- 권장 긴 변 1600px 이상
- 웹용 JPG 또는 PNG
- 너무 작은 캡처 이미지 금지
- 텍스트가 이미지 안에 박혀 있는 파일은 가능하면 지양
- 동일 섹션 이미지는 색감과 톤을 최대한 통일

---

## 7. 우선순위

필수 우선순위는 아래와 같다.

1. Hero 이미지와 Hero 카피
2. Brand / Coffee / Space 핵심 문구
3. 주소 / 운영시간 / 인스타그램 링크
4. Space 이미지 3장
5. 나머지 Brand / Coffee / Store 이미지

---

## 8. 전달 형식 권장

아래 방식 중 하나로 전달하면 된다.

- 텍스트: Notion / Google Docs / 메신저 본문
- 이미지: 압축 파일 또는 공유 드라이브 폴더
- 파일명은 권장 규칙 준수

권장 패키지 예시:

```txt
/barracuda-content
  /images
    hero.jpg
    brand-1.jpg
    brand-2.jpg
    brand-3.jpg
    coffee-1.jpg
    space-1.jpg
    space-2.jpg
    space-3.jpg
    store-1.jpg
    store-2.jpg
    store-3.jpg
  content.txt
```

---

## 9. 전달 후 반영 위치

개발 반영 위치는 아래와 같다.

- 텍스트 수정: `data/site-content.ts`
- 이미지 교체: `public/images/`

즉 디자이너가 이 문서 기준으로 자료를 전달하면,
개발자는 구조를 바꾸지 않고 콘텐츠만 교체하면 된다.
