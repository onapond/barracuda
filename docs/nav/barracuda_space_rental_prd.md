
# Barracuda Space Rental Platform — Codex PRD
Version: 1.0

Purpose
This document specifies the Space Rental page for the Barracuda website.
It is written for Codex to implement the page without additional clarification.

Barracuda is positioned as:
- Roastery Cafe
- Cultural Gathering Space
- Private Event Venue

This page must present Barracuda as a place where curated events can happen.

---

# 1. Page URL
/space

---

# 2. Page Goal
1. Present Barracuda as a premium spatial environment
2. Show types of events that can happen
3. Display the atmosphere through images
4. Provide a simple event inquiry form

The page must feel like a venue presentation, not a booking system.

---

# 3. Page Structure
Space Hero
Space Overview
Event Types
Gallery
Rental Information
Inquiry Form

---

# 4. Space Hero
Title: Barracuda Space
Subtitle: A space for gatherings, culture, and experience.

Description:
Barracuda is designed as a calm, curated environment where coffee,
culture, and people come together.

CTA: Host Your Event

---

# 5. Space Overview
Explain what the space represents.

Themes:
Coffee gatherings
Creative meetings
Cultural moments

Tone: calm and editorial.

---

# 6. Event Types
Examples:
Private Party
Fashion Show
Book Club
Workshop
Brand Event
Creative Meetup

Each card includes:
Title
Short description
Optional image

---

# 7. Gallery
Display 6–9 interior images.

Layout
Desktop: 3-column grid
Mobile: single column

Purpose: let visitors imagine events in the space.

---

# 8. Rental Information
Sections:
Capacity
Available hours
Equipment availability
Location

Keep information concise.

---

# 9. Inquiry Form
Fields:
Name
Phone
Email
Event Type
Event Date
Expected Guests
Message

Submit Button: Send Inquiry

MVP may send email or later connect to Supabase.
No payment required.

---

# 10. Design Principles
Style:
Minimal
Premium
Calm
Spatial

Avoid:
Booking-system UI
Crowded layouts
Bright colors
Heavy animations

---

# 11. Technical Requirements
Framework: Next.js
Language: TypeScript
Styling: TailwindCSS
Deployment: Vercel
Future backend: Supabase

---

# 12. Success Criteria
• inquiries submitted
• visitors spending time on gallery
• strong spatial impression

This page must feel like a venue presentation rather than a booking system.
